1. ¿Qué es el DOM?

El Document Object Model (DOM) es una interfaz de programación de aplicaciones (API) para documentos HTML y XML. Es una representación de la página web que permite a los programas acceder y manipular el contenido y la estructura de la página.

2. ¿Qué es un nodo?

En el DOM, un nodo es cualquier objeto que se encuentre en el árbol de nodos que representa la estructura de la página web. Cada elemento HTML, atributo, texto, comentario, etc., es un nodo en el árbol de nodos del DOM.

  • Elemento HTML: <p>, <div>, <h1>, etc.
  • Atributo: class, id, href, etc.
  • Texto: cualquier texto dentro de un elemento HTML
  • Comentario: <!-- comentario -->

3. ¿Cómo se accede a un elemento del DOM?

Hay varias formas de acceder a un elemento del DOM, pero las más comunes son:

  • getElementById(): devuelve el elemento que tenga el ID especificado
  • getElementsByClassName(): devuelve una lista de elementos que tengan la clase especificada
  • getElementsByTagName(): devuelve una lista de elementos que tengan el nombre de etiqueta especificado
  • querySelector(): devuelve el primer elemento que coincida con el selector CSS especificado
  • querySelectorAll(): devuelve una lista de elementos que coincidan con el selector CSS especificado

5. ¿Cuál es la diferencia entre getElementby* y querySelector*?

Las funciones que empiezan con "getElementBy" devuelven una lista de elementos que coinciden con el criterio de búsqueda especificado (ID, clase, etiqueta), mientras que las funciones que empiezan con "querySelector" devuelven solo el primer elemento que coincida con el selector CSS especificado.

6. ¿Qué es un evento?

Un evento es cualquier acción que sucede en una página web, como hacer clic en un botón, mover el ratón, escribir en un campo de texto, etc. Los eventos pueden ser detectados por JavaScript y se pueden utilizar para hacer que la página web responda a la interacción del usuario.

7. ¿Qué es un listener?

Un listener es una función de JavaScript que se utiliza para detectar un evento específico en un elemento del DOM. Cuando el evento ocurre en el elemento, la función se ejecuta automáticamente. Los listeners se pueden agregar a los elementos del DOM utilizando métodos como addEventListener() o onclick.

8. ¿Qué es un callback?

Un callback es una función que se pasa como argumento a otra función y que se ejecuta cuando se cumple una determinada condición o evento. En JavaScript, los callbacks se utilizan a menudo con funciones asíncronas, como las que se utilizan para hacer solicitudes HTTP o para esperar a que se carguen los recursos de una página web. Cuando la función asincrónica termina, llama al callback con los datos que se deben procesar.

9. ¿Qué es un objeto de evento?

Un objeto de evento es un objeto que se crea cuando se produce un evento en una página web. Este objeto contiene información sobre el evento, como el tipo de evento, el elemento en el que se produjo, las coordenadas del ratón, las teclas que se presionaron, etc. Los objetos de evento se pasan a las funciones de listener como argumentos, para que puedan acceder a esta información.
